# 个人网站 Prompt 存档

> 记录在构建和维护 https://hanaaa-sying.github.io/memyslf/ 过程中，反复使用的高效 Prompt 模板。

**新增条目格式：** 每次发现一个值得复用的 Prompt 后，在文档最末尾追加：

```
---

### YYYY年MM月DD日 — 一句话概括该 Prompt 的用途

**场景：** ...
**Prompt：**
（完整内容，可直接复制使用）
```

---

## 一、添加新的项目经历（简历区）

```
我想在简历的项目经历区新增一条经历，信息如下：

- 项目名：[项目全称]
- 性质：[课程项目 / 实习 / 竞赛 / 其他]
- 角色：[负责人 / 队员 / 创始团队成员 / ...]
- 时间：[YYYY.MM — YYYY.MM]
- 主要做了什么（bullets）：
  · [bullet 1]
  · [bullet 2]
  · [bullet 3]

有没有对应的作品集跳转？[有，id 为 xxx / 暂时没有]
```

**注意**：必须要求同时输出中英文对照，确认后再写入文件。

---

## 二、添加新的作品集卡片（Portfolio 区）

```
我想在作品集里新增一张卡片，信息如下：

- id：[唯一英文 id，如 "milan" / "yacht"]
- 中文标题：[...]
- 英文标题：[...]
- 中文简介（desc）：[一两句话描述项目]
- 英文简介（desc）：[...]
- 标签（tags）：[数据分析 / 咨询 / Claude Code / ...]
- 详情弹窗内容：
  · 有没有外链（Dashboard / 报告 / 论文）？链接：[...]
  · 有没有 PDF 文件？文件名：[...]
  · 有没有 PPT？文件名：[...]（注意先把文件放进 files/ 目录）
  · 弹窗里还需要展示哪些内容？[核心发现 / 图片 / 迭代故事 / ...]
```

## 三、调整卡片顺序

```
请把作品集里的 [xxx 项目卡片] 移到第 [1 / 2 / ...] 位。
中英文都要同步调整。
```

## 四、修改标题 / 描述

```
请把作品集里 id 为 [xxx] 的卡片：
- 中文标题改为：[新标题]
- 英文标题改为：[新标题]

（如果只改中文或只改英文，说明理由）
```

## 五、给作品集卡片添加链接

### 5a — 添加可在线浏览的外链（推荐）
```
请给 id 为 [xxx] 的卡片弹窗添加一个按钮：
- 按钮文字：[查看 Dashboard → / PPT → / 完整报告 →]
- 链接：[https://...]
- 在新标签页打开（target="_blank"）
```

### 5b — 添加本地文件下载链接
```
请给 id 为 [xxx] 的卡片弹窗添加一个下载按钮：
- 按钮文字：PPT ↓ / 报告 ↓
- 文件路径：files/[文件名]
- 用下载模式（download 属性），不用在线预览
```
> 注意：文件必须先放进 `files/` 目录，否则 GitHub Pages 找不到。

---

## 六、双语同步通用要求

每次提出内容更新时，都可以在末尾加上这句话，避免遗漏：

```
请同时输出中英文对照内容，我确认表达合适后再写入文件。
```

---

## 七、检查网站是否部署成功

```
我刚推送了更新，请帮我确认：
1. git push 是否成功？
2. 如果浏览器看不到更新，可能是缓存问题——请提醒我按 Ctrl+Shift+R 强制刷新。
3. 如果想看部署状态，去 GitHub 仓库 → Actions 标签页查看。
```
